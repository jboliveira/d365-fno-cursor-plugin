# Form Patterns Guide

## Selection Flow

1. Review metadata (Form.Design.Style, control names, data sources)
2. Run form and review visuals
3. Apply pattern via designer (Apply pattern on Design node)

## Common Patterns

| Pattern | Use when |
|---------|----------|
| SimpleListDetails | Grid + detail panel |
| DetailsFormTransaction | Header + lines |
| Operational Workspace | Role-based workspace (preferred over legacy Workspace) |
| Wizard | Multi-step data collection |
| Task | Single-purpose dialog |

## Avoid

- Custom pattern unless extensible controls require it
- Monster all-in-one forms copying legacy UX
